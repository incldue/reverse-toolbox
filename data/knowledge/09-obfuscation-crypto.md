# 混淆、编码与小型密码题

## 变换链思维

多数 CTF 逆向题可以抽象为：

```text
input -> transform_1 -> transform_2 -> ... -> compare(expected)
```

目标是恢复每一步的顺序、参数和边界。不要只看单个函数名。

## 常见编码

| 类型 | 特征 |
| --- | --- |
| Hex | 字符集 `0-9a-f`，长度通常为偶数 |
| Base64 | 字符集含 `A-Z a-z 0-9 + / =` |
| Base32 | 常见大写字母和 `=` |
| URL 编码 | `%xx` |
| XOR | 同一 key 重复、明密文异或后有周期 |
| ROT | 字母范围内平移 |

## 常见位运算

- `xor`：可逆，常与固定 key 或滚动 key 组合。
- `rol/ror`：循环移位，常见 hash 和轻量加密。
- `add/sub`：字节溢出后模 256。
- `swap`：置换数组或打乱索引。
- `crc`：通常有查表和多项式。

## 约束求解

适合 Z3 的场景：

- 每个字符有范围限制。
- 大量线性关系、异或、加法。
- 分支较少，表达式可完整抽取。

不适合直接 Z3 的场景：

- 大量哈希不可逆。
- 压缩、复杂 block cipher。
- 运行路径高度依赖输入且状态爆炸。

## 脚本复现模板

```python
def u8(x):
    return x & 0xff

def rol8(x, n):
    return u8((x << n) | (x >> (8 - n)))

def transform(data):
    out = []
    for i, b in enumerate(data):
        b ^= 0x23
        b = rol8(b, 3)
        b = u8(b + i)
        out.append(b)
    return bytes(out)
```

如果环境没有 Python，也可以用 Node.js 快速写同样逻辑。

## 处理混淆

| 混淆 | 处理 |
| --- | --- |
| 字符串加密 | 找解密函数，dump 运行时明文 |
| 控制流平坦化 | 找状态变量和 switch 分发 |
| 花指令 | 确认真实控制流，必要时 patch |
| API hash | 还原 hash 算法并枚举导出表 |
| 表驱动算法 | 提取表，写脚本验证 |

## 验证原则

- 每恢复一步就用样例输入验证。
- 记录中间值，避免最终失败才回溯。
- 如果脚本输出与调试器不同，优先检查字符集、符号扩展、大小端和溢出。
