# 赛场速查清单

## 逆向前 5 分钟

- [ ] 记录哈希、文件格式、位数、架构。
- [ ] 查壳、熵、入口点、导入表。
- [ ] 跑字符串和资源搜索。
- [ ] 判断是 native、.NET、Java、Android 还是脚本封装。
- [ ] 找到至少一个输入点或输出字符串引用。

## 静态分析

- [ ] 成功定位 main 或等价业务入口。
- [ ] 找到用户输入进入的位置。
- [ ] 找到成功/失败分支。
- [ ] 找到最终比较对象。
- [ ] 关键函数已命名，关键表已导出。
- [ ] 反编译结果已用汇编核对。

## 动态分析

- [ ] 在输入读取点断下。
- [ ] 在比较点断下。
- [ ] 记录比较两侧 buffer。
- [ ] 记录关键变换前后数据。
- [ ] 如有壳，确认 OEP 和 dump 是否可重新打开。

## 写 solver

- [ ] 明确输入长度和字符集。
- [ ] 明确每一步变换顺序。
- [ ] 处理字节溢出和大小端。
- [ ] 用测试输入对齐调试器中间值。
- [ ] 在原程序中验证 solver 结果。

## 壳与保护

- [ ] 检查 UPX 标准解包。
- [ ] 检查 TLS Callback。
- [ ] 检查自解密和运行时 API 解析。
- [ ] 如需 patch，记录原字节和新字节。
- [ ] dump 文件与原始样本分开保存。

## Android / .NET

- [ ] Android 已检查 Manifest、资源、Java 层和 native 库。
- [ ] JNI 已检查导出函数和 RegisterNatives。
- [ ] .NET 已检查资源、反射和混淆。
- [ ] 反编译失败处已回看 IL/smali/native。

## 提交前

- [ ] flag 格式正确。
- [ ] 不是调试器里误读的临时值。
- [ ] 能从干净样本复现。
- [ ] solver 和关键步骤已保存。
